# Эпик 14 — мультитенант и триал («AI-директор» для чужих владельцев)

> Восстановлен 2026-07-20: оригинал, записанный в сессии 14a, не сохранился на диске
> (вероятно, сбой синхронизации OneDrive). Содержание собрано из STATE.md, архива и
> решений владельца.

## Зачем

Владелец хочет дать ОДНОМУ доверенному тестеру бесплатный доступ на его собственном
канале. Разбор показал: полный Шаг 14 роадмапа (веб-дашборд + биллинг + API) для этого
не нужен — касса бессмысленна без платящих, сайт избыточен (продукт целиком живёт в
Telegram). Реальный блокер — мультитенант (хвост Шага 9): до эпика бот был одно-арендный.

## Решения владельца

- Полный Шаг 14 делаем как эпик, но в порядке «сначала триал, потом касса».
- Триал — ТОЛЬКО через Telegram, БЕЗ сайта (веб-кабинет 14c–14d отложен).
- Тестер пользуется существующим общим ботом (свой bot-token на клиента = 14b-bis, не сейчас).
- Ритм: 1 сессия = 1 подшаг.

## Безопасность (обязательный пункт)

Переключатель каналов берёт канал из нажатой кнопки — нужно проверять принадлежность
текущему владельцу, иначе крафтнутый callback выберет чужой. ✅ Закрыто в 14b-1
структурно: индекс в списке `listChannelsByOwner` + `pickSelectedId` отбрасывает id вне
списка. В 14b-2 такой же принцип применить к approval-callback'ам (`ap:*` несут id
`PendingPost` — проверять, что пост принадлежит каналу владельца).
