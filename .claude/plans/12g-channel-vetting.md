# Шаг 12g — Вет чужих каналов перед закупкой рекламы

Экран «🔍 Проверка канала»: владелец вводит `@username` чужого канала → бот тянет данные
Telemetr и выдаёт автоматический вердикт первого прохода 🟢/🟡/🔴 (порт чек-листа скилла
`telemetr-vetting` в объёме, доступном HTTP-API). Закрывает хвост эпика 12 (помечен кандидатом
с 12f). Переиспользует адаптер Telemetr (12e) и ядро аномалий (12f).

## Зачем
Главная боль проекта (из скилла): в эзо-нише реклама часто идёт во **взаимопиар/зодиак-сетки** —
трафик есть, оплат нет. Ручной вет по 6 шагам Башлыкова требует браузера Telemetr. 12g даёт
владельцу быстрый **первый фильтр прямо в боте**: отсеять явную сетку/накрутку/падение до того,
как тратить время на глубокий разбор. Это НЕ замена ручного вета — это дешёвое «сразу видно, что
не брать».

## Что доступно из HTTP-API (проверено на эталонах скилла)
`GET /channels/stat` отдаёт больше, чем читал адаптер 12e. Калибровка на трёх якорях скилла:

| сигнал | поле API | sofia (свой) | favoritehoro 🟢 | ezoterik_dnevnik 🔴 |
|---|---|---|---|---|
| подписчики | `participants_count` | 401 | 37046 | 56282 |
| охват/подписчик | `avg_post_reach`/subs | 7.2% | 12.6% | 3.4% |
| ER | `err_percent` | 5.64 | 5.62 | 2.45 |
| **сеть упоминаний** | `mentioning_channels_count` | 9 | 20 | **725** |
| оценка качества | `scoring_rate` (0–10) | 6.7 | 6.9 | 6.1 |

`mentioning_channels_count` — **решающий детектор сетки**: 20 у органики против 725 у зодиак-фермы.
Это порт «Шага 4» скилла (главный шаг) в объёме, который видит API (число каналов, но не сама
таблица клонов).

`GET /channels/subscribers` (ряд по дням, 28д) → тренд (падение месяц-к-месяцу) + аномалии (12f).

Чего API НЕ видит (остаётся ручному вету через скилл/браузер): имена зодиак-клонов в таблице
упоминаний, история описаний (перевёртыш), тон/этика контента, SCAM-плашка, пол аудитории.
Экран это честно проговаривает.

## Сигналы вердикта (пороги — именованные константы, niche-agnostic)
- **reachRatio** = `avg_post_reach / participants_count`: ≥5% 🟢 · 3–5% 🟡 · <3% 🔴
  (пороги evaluate_channel скилла; <3% = накрутка/мёртвая аудитория). **Решающий** (hard-red).
- **network** = `mentioning_channels_count`: ≤50 🟢 · 51–150 🟡 · >150 🔴 (сетка/ферма).
  **Решающий** (hard-red) — Шаг 4 скилла «главный».
- **err** = `err_percent`: 5–35 🟢 · <5 🟡 (вялая вовлечённость) · >35 🟡 (подозрительно/интерактивы).
- **scoring** = `scoring_rate`: ≥6 🟢 · 4–6 🟡 · <4 🔴.
- **trend** = `delta28d` ряда: ≥0 🟢 · <0 🟡 (падение — скилл: «снижение месяц к месяцу»).
  Нет ряда → сигнал опускается.
- **anomalies** = число резких скачков (`detectSubscriberAnomalies` 12f): 0 🟢 · ≥1 🟡
  (накрутка/рекламные всплески). Нет ряда → опускается.

## Правило вердикта (порт рубрики скилла «🔴 ≥2 красных флага»)
- есть хоть один **решающий** 🔴 (network=ферма ИЛИ reachRatio=накрутка) → 🔴;
- иначе reds ≥ 2 → 🔴;
- reds == 1 → 🟡;
- reds == 0, но есть 🟡 → 🟡;
- всё 🟢 → 🟢.

Проверка на якорях: ezoterik (network 725 = решающий) → 🔴 ✅; favoritehoro (все 🟢) → 🟢 ✅.

## Решения (зафиксированы)
- **Где**: кнопка «🔍 Проверка канала» на экране «📈 Рост» (вет = активность роста через рекламу).
- **Кэш**: НЕТ. Чужой канал не имеет строки `Channel` в нашей БД, вет — по требованию и редкий
  (владелец жмёт вручную перед закупкой). 2 запроса Telemetr на проверку (стата + ряд). Лимит
  10k/мес это легко держит; экран отмечает, что тратит запросы.
- **Провайдер**: строим из `deps.telemetrApiKey` (как `renderGrowth`). Нет ключа → «проверка
  недоступна».
- **Ввод**: pending `{ kind: "vetChannel" }`; принимаем `@username`, `t.me/x`, `telemetr.me/@x` —
  нормализуем до `@username`.
- **Приватный/несуществующий**: API вернул null → понятное сообщение, не падение.
